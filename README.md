# Менеджер задач

## 1. Идея проекта

Консольное приложение на Python для ведения задач с приоритетами, сроками и отметкой выполнения. Данные хранятся в CSV-файле средствами стандартной библиотеки Python.

Версия: `1.0.5`.

## 2. Пользовательские истории

1. Как студент, я хочу добавлять задачу с названием, описанием, приоритетом и сроком, чтобы не забывать учебные дела.
2. Как студент, я хочу видеть список всех задач, чтобы понимать текущую нагрузку.
3. Как студент, я хочу изменять задачу, чтобы исправлять название, срок, описание, статус или приоритет.
4. Как студент, я хочу удалять ненужные задачи, чтобы список оставался актуальным.
5. Как студент, я хочу искать задачи по названию и описанию, чтобы быстро находить нужные записи.
6. Как студент, я хочу фильтровать задачи по статусу и приоритету, чтобы отдельно видеть важные или выполненные дела.
7. Как студент, я хочу видеть просроченные задачи, чтобы вовремя исправлять план.

## 3. MVP

Обязательный минимальный продукт:

- создание задачи;
- просмотр всех задач;
- редактирование задачи;
- удаление задачи;
- отметка задачи выполненной;
- поиск по названию и описанию;
- фильтр по статусу и приоритету;
- хранение данных в CSV-файле;
- модульные тесты на CRUD, поиск, фильтры и ошибки ввода.

## 4. Структура данных

Файл хранения: `tasks.csv`.

| Поле | Тип | Описание |
|---|---|---|
| `id` | int | Уникальный положительный идентификатор |
| `title` | str | Название задачи, обязательное поле |
| `description` | str | Описание задачи |
| `priority` | str | `low`, `medium`, `high` |
| `due_date` | str | Срок выполнения в формате `YYYY-MM-DD` |
| `status` | str | `todo` или `done` |

## 5. Архитектура

Схема взаимодействия:

`пользователь → консольное меню → TaskManager → CsvTaskStorage → tasks.csv`

Структура проекта:

```text
task_manager_project/
├── main.py
├── task_manager/
│   ├── __init__.py
│   ├── cli.py
│   ├── models.py
│   ├── service.py
│   └── storage.py
├── tests/
│   ├── test_models.py
│   ├── test_service_crud.py
│   ├── test_service_filters_search.py
│   ├── test_service_regressions.py
│   └── test_storage.py
├── README.md
├── TEST_CHECKLIST.md
├── SUPPORT_LOG.md
├── REPORT.md
├── pyproject.toml
└── requirements.txt
```

Основные функции:

- `create_task` — добавить задачу;
- `list_tasks` — прочитать список задач с фильтрацией;
- `get_task` — получить задачу по ID;
- `update_task` — обновить задачу;
- `delete_task` — удалить задачу;
- `mark_done` — отметить выполненной;
- `reopen_task` — вернуть в статус `todo`;
- `search_tasks` — поиск по названию и описанию;
- `list_overdue` — просмотр просроченных задач.

## 7. TDD-подход

Разработка велась по циклу:

`Red → Green → Refactor`

Порядок:

1. тесты для модели и валидации;
2. тесты для чтения и записи CSV;
3. тесты для CRUD-операций;
4. тесты для поиска и фильтрации;
5. регрессионные тесты по найденным багам;
6. CLI после готовности сервисных функций.

## 10. Ограничения

- Язык: Python.
- Интерфейс: консольный.
- Хранение: CSV.
- Для хранения не используются внешние библиотеки.
- Основная логика вынесена в функции и классы, поэтому покрывается тестами.
